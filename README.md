## 🌟 About SnapPDF

<!-- Hero Illustration -->
<p align="center">
  <img src="public/images/generated/hero-illustration.png" alt="SnapPDF Hero — Document Processing Workspace" width="100%" style="max-width: 800px; border-radius: 12px;" />

SnapPDF is a **one-stop document powerhouse** built for Indian users. Whether you're compressing a PDF for a government portal, resizing a passport photo for Aadhaar, or generating a QR code for your business — SnapPDF handles it all.

> 💡 **The big difference?** Everything runs **inside your browser**. Your files never leave your device. No servers, no uploads, no tracking. Just fast, private, powerful document processing.

### 🎯 Why Users Love SnapPDF

- 🛡️ **100% Private** — Files processed locally in your browser. Zero uploads. Zero tracking.
- ⚡ **Blazing Fast** — Compress a 5MB PDF to 200KB in under 3 seconds. No upload/download wait.
- 📱 **Works Everywhere** — Phone, tablet, or desktop. Fully responsive design.
- 🌐 **Works Offline** — Core tools function without internet after the first load.
- 🔓 **No Login Required** — Open and use instantly. No account, no email, no signup.
- 💰 **Free Forever** — No hidden charges, no premium tiers, no watermarks. Ever.
- 🎨 **Light & Dark Theme** — Beautiful interface with glassmorphism, spring animations, and modern micro-interactions.
- ♿ **Accessible** — Keyboard navigable, screen reader friendly, semantic HTML.
- 🔍 **Command Palette** — Press `⌘K` / `Ctrl+K` to instantly search and jump to any tool.
- 🤖 **Smart File Router** — Drag & drop any file anywhere — auto-detects type and suggests the best tool.
- ✨ **Modern UI/UX** — Glass cards, animated gradients, spring physics, blob backgrounds, stagger animations.

## 🇮🇳 India-Specific Features

SnapPDF is **purpose-built for Indian users** — every preset, every size target, every standard is tuned for Indian government and daily use.

### 📋 Government Form Presets

| Preset | Size | Use Case |
|--------|------|----------|
| 🪪 **20 KB** | 20 Kilobytes | Passport photos & signatures for UPSC, SSC, State PSC online forms |
| 📷 **50 KB** | 50 Kilobytes | Aadhaar card updates, passport renewal photo uploads |
| 📄 **100 KB** | 100 Kilobytes | Document uploads for Income Tax, DigiLocker, various portals |

### 🛂 Passport Photo Standards

- 🪪 **Aadhaar** — 35 × 45 mm, white background, face covering 70-80% of frame
- 💳 **PAN Card** — 25 × 35 mm, white background, center-aligned face
- 🛂 **Indian Passport** — 35 × 35 mm (or 51 × 51 mm), white/light gray background
- 🏛️ **Government** — Flexible preset for any unspecified requirement

### 🧠 Smart Compression Engine

- 🎯 Targets **exact file sizes** required by government portals
- 🔄 Multi-pass algorithm: quality → dimensions → format change
- ⚡ Automatically selects JPEG for photos, PNG for transparent images

### 🔒 Privacy Compliance

- ✅ All processing happens **inside your browser** — client-side only
- ✅ Aadhaar, PAN, passport scans **never leave your device**
- ✅ No server logs of file names, sizes, or content
- ✅ Compliant with Indian Data Protection Act principles

## 🚀 Getting Started

### ✅ Prerequisites

- **Bun** v1.0+ → [Install Bun](https://bun.sh)
- **Node.js** v18+ (alternative runtime)

### 📦 Installation

```bash
# Clone the repo
git clone https://github.com/your-username/snapdf.git
cd snapdf

# Install dependencies
bun install

# Set up the database
bun run db:push

# Start the dev server
bun run dev
```

🎉 Open **http://localhost:3000** and you're ready to go!

### 📋 Available Commands

| Command | What It Does |
|---------|-------------|
| `bun run dev` | Start dev server with hot reload (port 3000) |
| `bun run build` | Production build |
| `bun run lint` | Check code quality with ESLint |
| `bun run db:push` | Push database schema to SQLite |
| `bun run db:generate` | Generate Prisma database client |

### 🔑 Environment Variables

```env
# Database
DATABASE_URL="file:./db.sqlite"

# Auth (optional)
NEXTAUTH_SECRET="your-secret"
NEXTAUTH_URL="http://localhost:3000"

# AI Features (optional)
Z_AI_API_KEY="your-api-key"
```

## 🚢 Deployment

### Deploy to Vercel (Recommended)

SnapPDF is ready for **one-click deployment** on Vercel. Since all file processing is client-side, there are no heavy server requirements.

#### Step 1: Push to GitHub

```bash
# Initialize git (if not already)
git init

# Add all files (.gitignore handles exclusions)
git add .

# First commit
git commit -m "feat: initial SnapPDF release"

# Create repo on GitHub, then:
git remote add origin https://github.com/YOUR_USERNAME/snapdf.git
git branch -M main
git push -u origin main
```

#### Step 2: Deploy to Vercel

1. Go to [vercel.com](https://vercel.com) and sign in with GitHub
2. Click **"Add New Project"**
3. Import your `snapdf` repository
4. Vercel auto-detects Next.js — no configuration needed!
5. Click **"Deploy"**

> Done! Your app will be live at `snapdf-xxx.vercel.app`

#### Step 3: Set Environment Variables (Optional)

In Vercel Dashboard → Settings → Environment Variables:

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | No | SQLite path (default: `file:./db/custom.db`) |
| `Z_AI_API_KEY` | No | AI features (passport photo background removal) |
| `NEXTAUTH_SECRET` | No | Auth system (future feature) |

> Since SnapPDF processes all files **client-side**, no env vars are required for the core 21 tools.

#### Step 4: Custom Domain (Optional)

1. In Vercel Dashboard → Settings → Domains
2. Add your domain (e.g., `snapdf.in`)
3. Update DNS records as instructed by Vercel
4. SSL certificate is auto-provisioned

### Vercel Configuration

| Setting | Value | Notes |
|---------|-------|-------|
| **Framework Preset** | Next.js | Auto-detected |
| **Build Command** | `npx prisma generate && next build` | Auto-set via `vercel.json` |
| **Node.js Version** | 20.x | Auto-detected from `package.json` |
| **Output Mode** | Standalone | Set in `next.config.ts` |

### CI/CD with GitHub Actions

A CI workflow is included at `.github/workflows/ci.yml` that runs on every push and PR:

- **Lint** — Runs ESLint to check code quality
- **Build** — Runs `next build` to verify no build errors

Both jobs must pass before merging.

### Local Production Build

```bash
# Install dependencies
bun install

# Build for production
bun run build

# Start production server
bun run start
```
